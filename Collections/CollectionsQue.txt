2Ô∏è‚É£ What are the key interfaces in the Java Collections Framework?

Collection: Root interface for groups of objects. Has methods like add(), remove(), size().

Subinterfaces:

List: Ordered collection, allows duplicates (e.g., ArrayList, LinkedList)

Set: No duplicates allowed (e.g., HashSet, TreeSet)

Queue: For holding elements prior to processing (e.g., PriorityQueue)

Map: Not a Collection, but stores key-value pairs (e.g., HashMap, TreeMap)

7Ô∏è‚É£ What is the internal structure of a HashMap?

Uses an array of buckets, where each bucket is a linked list or balanced tree (for many collisions).

Hashing: The key‚Äôs hashCode() is used to find the bucket index.

Collision: When two keys have the same hash, they are stored in the same bucket ‚Äî resolved by chaining (linked list or tree).

Since Java 8, if bucket size exceeds threshold, linked list converts to balanced tree (Red-Black tree) for better performance.

8Ô∏è‚É£ What‚Äôs the difference between equals() and hashCode()? Why are they important in collections?

equals() checks logical equality of objects.

hashCode() returns an integer hash, used for placing objects in hash-based collections (HashMap, HashSet).

Contract: If two objects are equal (equals() returns true), their hashCode() must be the same.

If not, collections like HashMap will malfunction (e.g., fail to find keys).

1Ô∏è‚É£1Ô∏è‚É£ What is ConcurrentHashMap? How is it different from HashMap?

ConcurrentHashMap is a thread-safe version of HashMap.

Allows concurrent read and write by multiple threads without locks on the entire map.

Internally uses segment locks or finer-grained locking.

HashMap is not thread-safe ‚Äî concurrent modification may cause data corruption.

1Ô∏è‚É£2Ô∏è‚É£ Can a HashMap have duplicate keys? What happens when you put the same key twice?

No duplicate keys allowed.

If you put a key that already exists, the old value is replaced with the new value.

Example:

map.put("A", 1);
map.put("A", 2); // replaces 1 with 2
